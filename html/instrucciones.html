Cartas Pokemon (1º DAW/DAM)

SdA: Análisis, diseño y desarrollo de WebApp que genera 6 cartas Pokemon con la PokeAPI

Instrucciones

Resumen de los entregables

Al final del proyecto, cada equipo debe entregar tres productos:

  1. Infografía interactiva de la fase de análisis.
  2. WebApp en un repositorio de GitHub con link público.
  3. Vídeo demo de 2-3 minutos.

La entrega formal se hace en el aula virtual con los enlaces a los tres productos.

Criterios de aceptación mínimos del proyecto

Para considerar el proyecto entregado:

  • La app muestra 6 cartas Pokémon con datos reales de la PokéAPI.
  • Cada carta tiene estilos diferenciados según el tipo del Pokémon.
  • La lógica shiny está implementada y se puede demostrar.
  • El código está en GitHub con commits significativos y al menos una rama por sprint.
  • Hay un README.md completo.
  • Se han entregado los tres productos: webapp, infografía y vídeo.

Ampliaciones opcionales

Para quien acabe antes y quiera más:

  • Desplegar la app en GitHub Pages.
  • Persistir las cartas favoritas en localStorage.
  • Añadir un botón de "regenerar" con animaciones.
  • Implementar animaciones CSS para la aparición de shiny (brillos, partículas).
  • Añadir efectos de sonido (cuidado con la accesibilidad: que se puedan desactivar).
  • Internacionalizar la app (ES / EN).
  • Implementar la lógica del proyecto también en Java como ejercicio paralelo (consumo de la API con HttpClient, parseo con Gson o Jackson).
